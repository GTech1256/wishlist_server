### **Основные фичи приложения**


#### 0. **Инициализация: получение данных для авторизации**
- **Описание**: Пользователь может добавить желание, указав название, описание, цену, ссылку на товар и загрузив изображение.
- **Дополнительно**:
  - Возможность указать приоритет желания (например, "очень хочу", "хочу", "может быть").
  - Категории желаний (например, "техника", "одежда", "книги").
  - Возможность добавить несколько изображений.
  - Поле для комментариев (например, "размер", "цвет").
- **Техническое описание**:
  ```json
  GET /user/auth/data
  initData -> authToken //Todo jwt???
  ```
  ```
  GET /user/stats
  authToken -> {
  WishTotal: number
  WishPublic: number
  WishComplete: number
  }
  ```

---


#### 1. **Добавление желания**
- **Описание**: Пользователь может добавить желание, указав название, описание, цену, ссылку на товар и загрузив изображение.
- **Дополнительно**:
  - Возможность указать приоритет желания (например, "очень хочу", "хочу", "может быть").
  - Категории желаний (например, "техника", "одежда", "книги").
  - Возможность добавить несколько изображений.
  - Поле для комментариев (например, "размер", "цвет").
- **Техническое описание**:
  ```json
  POST /wish/add
  FormData {
    title: string,
    description: string,
    price: number,
    link: string,
    image: Blob(File),
    priority?: string,
    category?: string,
    comments?: string
  } -> {
    id: string,
    title: string,
    description: string,
    price: number,
    link: string,
    image: string (url),
    priority?: string,
    category?: string,
    comments?: string
  }
  ```

---

#### 2. **Просмотр желания**
- **Описание**: Пользователь может просмотреть детали желания, включая автора, изображение, название, описание, цену и ссылку.
- **Дополнительно**:
  - Отображение статуса желания (например, "активно", "забронировано", "исполнено").
  - Кнопка "Забронировать" (если это публичное желание).
  - Кнопка "Добавить в мой список" (для копирования желания в свой список).
- **Техническое описание**:
  ```json
  GET /wish/{id}
  void -> {
    id: string,
    title: string,
    description: string,
    price: number,
    link: string,
    image: string (url),
    status: string,
    author: {
      name: string,
      nickname: string,
      avatar: string (url)
    }
  }
  ```

---

#### 3. **Личный список желаний**
- **Описание**: Пользователь может просматривать свои добавленные желания, редактировать их, делать публичными или закрытыми.
- **Дополнительно**:
  - Фильтрация по статусу (например, "активные", "забронированные", "исполненные").
  - Сортировка по дате добавления, цене, приоритету.
  - Пагинация (например, по 5 элементов на странице).
- **Техническое описание**:
  ```json
  GET /wish/user
  void -> Array<{
    id: string,
    title: string,
    description: string,
    price: number,
    link: string,
    image: string (url),
    isPublic: boolean,
    status: string,
    priority?: string,
    category?: string
  }>
  ```

---

#### 4. **Поиск по публичным желаниям**
- **Описание**: Пользователь может искать публичные желания других пользователей.
- **Дополнительно**:
  - Фильтры по категориям, цене, приоритету.
  - Пагинация и сортировка.
- **Техническое описание**:
  ```json
  GET /wish/search?search={string}&offset={0}&category={string}&minPrice={number}&maxPrice={number}
  void -> Array<{
    id: string,
    title: string,
    description: string,
    price: number,
    link: string,
    image: string (url),
    author: {
      name: string,
      nickname: string,
      avatar: string (url)
    }
  }>
  ```

---

#### 5. **Личный кабинет**
- **Описание**: Пользователь может просматривать свою статистику (аватар, ник, количество желаний и т.д.).
- **Дополнительно**:
  - Возможность редактировать профиль (аватар, ник, описание).
  - Отображение активности (например, "забронировано ваших желаний: 5").
- **Техническое описание**:
  ```json
  GET /profile/{id}
  void -> {
    name: string,
    nickname: string,
    avatar: string (url),
    wishTotal: number,
    wishPublic: number,
    wishComplete: number,
    bio?: string
  }
  ```

---

#### 6. **Поиск пользователей**
- **Описание**: Пользователь может искать других пользователей по нику и просматривать их списки желаний.
- **Дополнительно**:
  - Возможность добавить пользователя в друзья.
  - Отображение общих друзей.
- **Техническое описание**:
  ```json
  GET /user/search?nickname={string}
  void -> Array<{
    name: string,
    nickname: string,
    avatar: string (url),
    wishCount: number
  }>
  ```

---

#### 7. **Подарочный список**
- **Описание**: Пользователь может создать список с дедлайном, добавить в него желания и поделиться ссылкой для бронирования.
- **Дополнительно**:
  - Уведомления о бронировании (например, "Ваше желание забронировано").
  - Возможность указать бюджет на список.
  - Статус списка (например, "активен", "завершен").
- **Техническое описание**:
  ```json
  POST /giftlist/create
  {
    title: string,
    deadline: string (date),
    wishes: Array<string> (wish IDs),
    budget?: number
  } -> {
    id: string,
    link: string (url)
  }
  ```

---

#### 8. **Друзья**
- **Описание**: Пользователь может добавлять друзей и просматривать их желания.
- **Дополнительно**:
  - Уведомления о новых желаниях друзей.
  - Возможность комментировать желания друзей.
- **Техническое описание**:
  ```json
  POST /friends/add
  {
    userId: string
  } -> {
    status: string
  }
  ```

---

#### 9. **Бронирование желаний**
- **Описание**: Пользователь может бронировать желания из публичных списков или подарочных списков.
- **Дополнительно**:
  - Ограничение на количество бронирований (например, не более 2 раз).
  - Уведомление автору о бронировании.
- **Техническое описание**:
  ```json
  POST /wish/reserve
  {
    wishId: string,
    userId: string
  } -> {
    status: string
  }
  ```

   - Название списка, дедлайн, список желаний.
   - Кнопка "Поделиться".
